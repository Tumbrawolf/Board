# Playtest Log — Full Game Walkthrough (2026-06-24)

Running log of rules gaps, contradictions, and decisions found while playing a full 4-player Normal-difficulty game start-to-finish, narrated round by round. Numbering continues from README.md's Feedback list (last entry there was #30). **Nothing in this file has been applied to `Rules.docx` / `Board_Rules_Reorganized.docx` yet** — it's a holding pen for end-of-game review, per explicit instruction this session.

Setup: 4 players (all seats played by the user + assistant), Normal difficulty, Commander's Call optional rule in use. Deck seed 7 (deterministic draws via a scratch dealer script, not checked into the repo).

---

31. **Commander Actions (Planning) vs. Worker Placement ordering gap.** Planning Stage lists Commander Actions (spend command resources, delegate extra lanes, draw/choose the round's Event, roll for Boss spawn, draw a Mission Assignment pool sized by "the commander's Rank") *before* Worker Placement — but the commander is defined elsewhere (4.5 Command, 7.2 Key Terms) as "the 1st worker placed at Command each round," which only happens during Worker Placement. As written, you can't perform Commander Actions without already knowing who the commander is, yet that's not determined until the later step.
    - **Resolution decided this session:** the commander for a round's Commander Actions is a **holdover from the previous round** — whoever was commander last round keeps running Commander Actions at the top of this round, and this round's Worker Placement (the Command race) determines who becomes commander for *next* round's Commander Actions. Round 1 has no previous round, so the **Leader fills in as Round 1's commander** by default.
    - Not yet written into either rules doc. When applying: needs a sentence in 4.5 Command and/or 7.2 Key Terms' "Commander" definition clarifying the holdover, plus a Round 1 exception note pointing at Leader Selection.

32. **Command Card hand refill happens at Cleanup, not Planning, despite what the current docx text says.** `Rules.docx`'s Planning Stage / Commander Actions still literally reads "Any players with less than 2 command cards must draw until they hold 2 cards. This number is 3 for commander" — but this was already corrected in a prior session (2026-06-24, see project memory): refill (discard unwanted cards to the bottom of the deck, redraw to hand size — 2 normal, 3 for that round's commander) happens at **Cleanup**, once the round's commander is already finalized, not at the top of Planning before anything else has happened. This is an established, confirmed design decision from before this playtest, just never written into the docx text. **Apply the Cleanup-stage version in all 3 games below** — do not follow the literal Planning-stage line in the current docx.

33. **Enemy Rank-pile supply shortfall — PROVISIONAL resolution, pending final user sign-off.** Found while attempting Round 1 Deployment of an earlier (now-restarted) walkthrough: the **Fodder** rank (Enemy Progress 0-1, covering all of Round 1 and likely Round 2) has only **2 unique cards** in the entire 93-card `Enemy Stats.csv` deck ("Pests," "Ticks"), but Normal-difficulty Round 1 alone needs 3 enemies/lane × 4 lanes = **12** Fodder-rank enemies dealt. The rules never say what happens when a Rank pile runs dry mid-deal. User was asked to pick a resolution but the conversation moved to this 3-game test before they answered. **Provisional call for these 3 games (use uniformly across all of them for consistency): treat each Enemy card as having unlimited physical copies available** — i.e., a Rank pile never actually "runs out," the same card can be dealt multiple times in the same round/game. Reasoning: simplest fix, requires no mid-round reshuffle logic, and Pests' own Passive ("+1 damage for each other revealed Pest") structurally implies the designer expected multiples of the same card to appear together. **This is provisional — confirm with the user after all 3 games wrap**, and revisit if a game's results seem to hinge heavily on it (e.g., if Pests/Ticks spam trivializes or dominates early rounds).

34. **Findings from the 3-game manual playtest batch (Games 1-3, 2026-06-25) — consolidated from each game's own "Open Rules Questions Found" section.** All 3 games played to genuine completion (all 3 ended in a **loss**: Game 1 in 4 rounds, Games 2 and 3 in 5 rounds each). Two findings were reached *independently* by separate games (same ruling, no cross-talk between the agents that played them) — treat those as higher-confidence signals that the ruling is the obviously-correct one, just not yet written down anywhere official:
    - **Empty Active slot / lane with no player unit at all → counts as Overrun.** Reached independently in both Game 2 and Game 3. "Overrun" is defined as "what happens to a lane when enemies survive combat there with no player unit left to defend it" — both games read a never-deployed lane as already satisfying that condition (no exchange to resolve, enemies automatically survive). Worth promoting to an actual Key Terms clarification given the double convergence.
    - **Base deck exhaustion is a dealer-tooling artifact, not a rules gap** — both the Unit deck (Games 2 and 3 independently) and, originally, the Enemy Fodder pile (#33 just above) ran dry mid-game because this playtest's draw tooling has no "skim past and put back" mechanism for Direct fill's Rank-1-3 search, nor a discard-return path for unsold/examined cards. All 3 instances were fixed the same way (reshuffle the full base list as a fresh pool and keep drawing — treat printed cards as unlimited physical copies). This is very likely fine for the *real* physical game (a human thumbing through a deck doesn't remove every card they skip past) but is a known limitation of how this digital playtest was run, not evidence the actual card counts are too small. Don't mistake this for a balance problem with deck sizes.

    One-off findings, by game:
    - **(Game 1) Pests' Passive ("+1 damage for each other revealed Pest") — board-wide or lane-local stacking, and does it accumulate across the round's full reveal order?** Ruled board-wide, accumulating in actual resolution order. Produced real swings (a 1-damage Fodder card hitting for 6). Needs an explicit scope ruling, especially since hoard size scales up with Player Progress (up to 5/lane at Normal — more Pests, bigger possible stacks).
    - **(Game 1) Does equipped Gear survive a unit's death in combat (as opposed to a deliberate Retire)?** Retire from Duty explicitly returns gear to hand; combat death is silent on this. Ruled gear is lost alongside its unit on combat death, only salvageable via an explicit Retire.
    - **(Game 1) Chain of Command Event ("all unit HP/Damage = their Rank total") — which Rank scale for enemies, who don't use the player Rank scale?** Ruled each side reads its own scale's tier number (player Rank vs. Enemy Rank Order), gear bonuses stacking on top of the overridden base. Also affects how the Event's "outrank enemies in each lane" completion condition compares the two different scales.
    - **(Game 1) Enemy Progress's Escalate step (Cleanup) raises the track *after* that same round's Deployment-Stage enemy draw already used the old value.** Confirmed this is the only internally consistent reading given the stage order (Escalate can't run before Deployment in the same round) — a round's own escalation only ever affects *next* round's draw. Worth stating explicitly since it's easy to misapply a round early.
    - **(Game 1) Enemy Scouting's "move the top card into Active" — if that card leaves Active again the same round without dying (a swap/retreat Reveal), does whatever promotes into the vacated slot trigger a fresh Reveal too, chaining indefinitely?** Ruled yes, chains resolve immediately and fully during the upfront Scouting step. Could plausibly also be capped at one promotion, or deferred into the Combat Cycle instead — needs a real ruling since several enemy cards have exactly this kind of swap/retreat Reveal.
    - **(Game 1) Do multi-target enemy Reveal effects ("damage to all lanes," "damage to the lowest-HP unit") reach the player side only, or can they also hit enemies / cross into lanes that already finished resolving combat this round?** Ruled player-side only, and reachable across any lane regardless of that lane's own resolution order — produced one of the most decisive single effects in Game 1 (a Reveal killing 2 full-HP units in lanes it wasn't even fighting in).
    - **(Game 1) Can a player claim a Mission whose bracketed Rank exceeds their own current Rank, once its objective text is already true?** Ruled no — bracketed Rank is a hard completion gate, not flavor text, consistent with Direct/Roll Fill's Rank-gating elsewhere.
    - **(Game 2) Only one Active-vs-Active exchange resolves per lane per round — a reserve unit (player OR enemy) promoted mid-round after its side's Active card dies does NOT get a fresh exchange the same round, only "ready for next round."** Applied symmetrically to both sides for consistency. Combat Cycle's "until one of them dies... move to next lane" text is genuinely ambiguous about this.
    - **(Game 2) Honorable Discharge's Round Effect ("Units Retire on death this round") — does it reach a Scout killed at the Battlefield's Scout Slot (never in a lane), or only lane-combat deaths?** Ruled lane-combat-only; the Scout's death got no retirement refund.
    - **(Game 2) Multi-target Reveal damage splitting order, when a Reveal hits both a lane's Active AND Reserve unit (Lancer) — which takes damage first, and does Splash overflow to the second target?** Ruled Active first, Splash overflow to Reserve — but the one case that came up had no Reserve present, so this is still untested in practice.
    - **(Game 2) A Failure Penalty that targets a resource/slot that doesn't exist yet (Research Drive: "disable a Containment Block slot" when 0 slots are unlocked) — no-op, or does it pre-emptively reduce the first future unlock?** Ruled no-op (gentler reading); the punitive alternative (first Containment Protocol purchase grants only 1 slot instead of 2) was considered and rejected as reaching too far from the card's literal text.
    - **(Game 2) Pacing observation (not a contradiction): the Gear deck's Rank distribution skews high enough that early Armory Direct-fills can take far longer (12 skipped cards in one instance) than early Barracks fills ever do under the identical rule.** Worth a look if early-game Gear access feels notably slower than Unit access at the table.
    - **(Game 2) Once the Overrun Tracker hits 0 partway through Cleanup's sub-steps (Overrunning Lanes is checked before Event Resolution/Promotions/Escalate), do the remaining sub-steps still run?** Ruled no — the loss is immediate, remaining sub-steps are skipped as moot. Game 2 ended without ever resolving its final round's Event, Promotions, or Escalate as a result.
    - **(Game 3) During a Direct fill search, cards above the Rank-1-3 ceiling get revealed/skipped before a valid pick is found — do they stay in the deck (preferred, a real physical search wouldn't remove them) or get set aside?** Flagged primarily as a dealer-tooling gap (see the deck-exhaustion finding above) rather than a rules gap, but the rules themselves never address what a commander does with an over-Rank card surfaced mid-search either.
    - **(Game 3) Does an Event's Failure Penalty referencing "at combat start" apply retroactively to the round just resolved, or only to next round's combat?** Ruled non-retroactive (a no-op for the round that already happened) since the card text doesn't specify.
    - **(Game 3) Can the commander promote themselves during Promotions?** Rule says "select another player" — ruled as excluding self-promotion, consistent with how Rules' own Exceptions section already flags this exact question as open.
    - **(Game 3) When a Boss targets a lane that still has its own enemy hoard, does that lane's normal Combat Cycle AND the separate Boss damage-exchange both happen, or does one replace the other?** Ruled independent — the lane fights its own hoard normally in the Combat Cycle, and separately exchanges one damage instance with the Boss the same round.
    - **(Game 3) Movement/positional enemy Reveals with no stated tiebreak (Lance Turret: "swap to the lane with the lowest-damage active unit") — how are ties broken, and what happens if the swap target is the lane it's already in?** Ruled lowest-lane-number tiebreak, and a same-lane "swap" simply does nothing. Both are reasonable defaults invented for this game, not written anywhere.
    - **(Game 3) Does a Secret Objective ending "...and your objective is now complete" (Deus Machina) fire once ever, or every time its trigger condition is met again later in the same game?** Ruled one-time-only, since an infinitely-repeatable version would let one card indefinitely prevent the team from ever losing. The card text doesn't explicitly say "once per game."
